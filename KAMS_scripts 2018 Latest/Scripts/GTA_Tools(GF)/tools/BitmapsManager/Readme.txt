	    Bitmaps manager 1.0
====================================================


Совместимость: 3ds max 2009 и выше (возможно и старее версии)

	Описание:
----------------------------------------------------
Скрипт позволяет произвести сбор всех текстур моделей в одну выбранную директорию.
При сборе текстур, для каждой модели будет создан свой отдельный подкаталог с названием, соответствующим имени модели.
В скрипте так же присутствует возможность вывести информацию и текстурах в текстовом виде в окно listener'a (запустить listener по умолчанию F11),
кроме того, в данное окно будет выводится важная информация о результате копирования текстур, в которой будет сообщено об ошибках или успешном выполнении операции.


	Установка:
----------------------------------------------------
Поместите скрипт в любую папку, например в ../3dmax_folder/Scripts/
Запускайте из меню программы 3dmax либо перетащите скрипт прямо в окно 3dmax


	Использование:
----------------------------------------------------
Выберете необходимые модели.
Для вывода информации о текстурах выбранных моделей в окно listener'a, нажмите кнопку "Print" в блоке "Print list bitmaps".

Что бы собрать все текстуры выбранных моделей, сначала укажите конечный каталог в блоке "Copy to path" в поле. Кнопка - "to path" откроет окно проводника, для выбора необходимой вам директории.
Далее нажмите кнопку "Copy bitmaps to path!" и дождитесь окончания выполнения скрипта. Время выполнения скрипта напрямую зависит от количества моделей и их текстур



	Описание сообщений в логе listener'a:
----------------------------------------------------

Структура сообщения при выводе текстовой информации о текстурах модели такова

	название модели
	----------------------------------
	bitmaps:
	- список текстур...
	MISSING bitmaps:
	- список текстур...

под заголовком "bitmaps:" идет список найденных текстур модели
под заголовком "MISSING bitmaps:" идет список текстур, которые небыли найдены и их пути не верные (обычно на таких моделях текстуры не отображаются)
(!) некоторая информация так же может быть не отображена в этих блоках.
	- - -

Структура сообщения при копировании текстур модели такова
	название модели
	----------------------------------
	bitmaps NOT FOUND:
	- список текстур...

log: error copying bitmaps to: с:\some_folder\texture.png (maybe the bitmap already exists)
- done

под заголовком "bitmaps:" идет список текстур, которые небыли найдены и их пути не верные
в строчках "log: error copying bitmaps to:..." отображены пути, куда скрипт не смог скопировать текстуру, возможно по причине уже присутствия этой текстуры по указанному пути
(!) некоторая информация так же может быть не отображена в этих блоках.
